系统地图

01 / OVERVIEW

框架把批量任务拆成配置流、执行流、控制流和观测流。四条流在 ScopedCallable 与 CancellationToken.lateBind 汇合。

实线是主执行路径；上下文和控制信息从父线程注入，检测结果在请求结束时回流。
CONFIG

配置流

formalized 把并行度收敛到 [1, taskSize]，无显式超时则使用默认值。

CONTROL

控制流

ThreadRelay 找到父令牌，新令牌通过 parent 建链，late-bind 再接到 futures。

OBSERVE

观测流

ScopedCallable 采集 submit/start/end，向 TaskListener 发事件。

一次 map 调用的主数据链

02 / MAINLINE

入口是同步编排动作，任务和补充提交是异步的。返回 AsyncBatchResult 不代表任务完成，只代表结果容器和提交控制已建立。

01 · RESOLVE

解析执行器

按名称取 ListeningExecutorService；不存在抛 IllegalArgumentException。

02 · NORMALIZE

规范化选项

空输入短路；有效输入得到名称、并行度、超时和任务类型。

03 · WRAP

包装任务

每个元素变成携带令牌、选项、执行器和提交时间的 ScopedCallable。

04 · RETURN

返回批次

窗口返回 futures；随后 late-bind 连接失败即止、超时和父级取消。

代码锚点：Par.map → executeParallel → submitAll。
Par.map → formalized → logTaskPair → new CancellationToken(parent) → ScopedCallable → submitAll → lateBind → AsyncBatchResult

上下文、令牌与取消

03 / CONTROL

取消是可传播的状态协议，不是直接杀线程。负数状态表示当前任务应尽快中止；任务在检查点或可中断阻塞点观察它。

两级 ThreadRelay

curMap 保存当前任务写入的令牌、选项、任务名和执行器名；TTL 把父线程 curMap 复制为子线程 parentMap。子任务只读取 parentMap，不会把兄弟任务的当前值当成父上下文。

四种取消来源

状态触发
FAIL_FAST兄弟 future 失败
TIMEOUTtimer 超时
MUTUAL手动 cancel
PROPAGATING父令牌取消
合作协议：ScopedCallable.call 自动检查一次；长时间 CPU 计算需主动调用 Checkpoints.checkpoint(name, true)。

04 / SCHEDULER

窗口限制同时在飞的任务数，不是输入列表长度。初始提交 parallelism 个，完成一个才补一个；补充循环运行在全局 submitter pool。

CPU_BOUND

拒绝入队

SmartBlockingQueue.offer 返回 false，触发拒绝策略；CPU 任务可回退 caller/direct executor。

IO_BOUND

允许等待

默认进入有界队列，适合远程调用或数据库等待，仍需 parallelism 和 timeout。

PURGE

超时清理

HeuristicPurger 按速率限制清理线程池中的取消任务。

维护检查清单

08 / OPERATIONS

修改 API

确认 formalized 默认值、空输入语义和 BatchReport 状态一致。

修改取消

先画清 futureToken、父 token、submitCanceller；负数状态必须可被检查点识别。

修改线程池

同时验证队列、拒绝策略和 caller fallback；CPU/IO 差异是契约。

验证入口

mvn test，重点关注 ParTest、CancellationTokenTest、TaskGraphTest。

边界：库提供协作式取消、失败即止和检测，不内置重试、容错聚合或响应式链式编排。
